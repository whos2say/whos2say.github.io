QA Test Runner

Photo Album — Automated + Manual Checks

Environment
Supabase Connection
HEIC Conversion Library
Audio URL Parsing

Manual QA Checks

1. HEIC Upload → Thumbnail

Upload a .HEIC file from an iPhone. After upload, the item should show a thumbnail preview.
If no thumbnail: check browser console for "HEIC conversion failed" errors. Test: upload.html

2. MP3 Audio Autoplay in Slideshow

In an album, open ♪ Music, paste a direct MP3 URL and save. Open the slideshow. Press Play. The 🔊 button should appear and audio should play (or show "🔊 (Click)" if browser blocked autoplay). Test: albums.html

3. YouTube Embed in Slideshow

Set a YouTube URL as music (e.g. https://www.youtube.com/watch?v=dQw4w9WgXcQ). Slideshow should show a small YouTube player in the top-right. The mute button should be hidden. Audio should play from the embed.

4. Spotify Embed in Slideshow

Set a Spotify track URL as music (e.g. https://open.spotify.com/track/4cOdK2wGLETKBW3PvgPWqT). Slideshow should show a small Spotify player in the top-right. Note: Spotify free accounts may not autoplay — a login prompt may appear.

5. Music Badge Persists on Reload

After saving a music URL, reload the album page. The ♪ Music button should show the ✓ Set badge immediately without opening the modal. Hovering the button should show the full URL in the tooltip.

6. HEIC Files Show in Album Grid

After uploading a HEIC file, go back to the album. The photo tile should show the converted JPEG image. If it shows a broken image icon, the file_path in DB may have a wrong extension.
